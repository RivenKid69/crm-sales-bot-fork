# Wipon Pro API

## Overview

Система представляет собой многослойное приложение, где каждый модуль отвечает за свою область. Данные проходят через слои, начиная с входных точек, обрабатываются бизнес-логикой и сохраняются в базе данных. Модули связаны через вызовы функций и передачу данных. Используются паттерны, такие как MVC и DDD, для организации кода и обеспечения масштабируемости.

## Architecture Patterns

MVC - используется в модуле 'presenter' для разделения логики отображения и бизнес-логики, DDD - применяется в модуле 'use-cases' для реализации доменных правил и операций, Repository - используется в модуле 'data' для абстракции доступа к базе данных
