# guard

Этот модуль отвечает за защиту API-маршрутов, обеспечивая доступ только авторизованным пользователям. Он проверяет наличие и корректность JWT-токена, используя данные из других модулей, чтобы определить, может ли пользователь получить доступ к определённым ресурсам. Модуль работает как 'стражник', который проверяет аутентификацию перед выполнением запроса.

## Responsibilities

- Проверяет наличие и корректность JWT-токена в заголовке HTTP-запроса
- Использует use-case для поиска пользователя по токену и подтверждения его действительности
- Разрешает или блокирует доступ к защищённым маршрутам на основе результата проверки
