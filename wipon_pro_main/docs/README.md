# Wipon Pro Frontend

## Overview

Система построена как многоэтажный дом, где каждый модуль — это отдельная комната с определённой функцией. API — это почтовый ящик, принимающий и отправляющий данные. Store — центральный сейф, хранящий все важные данные. Router — это пропускной пункт, проверяющий, кто может войти в определённые комнаты. Auth — это охранник, проверяющий документы. Account и Subscriptions — это кабинеты, где хранятся личные данные и подписки. Utils — это мастерская, где делают полезные инструменты. Wipon-pro-main — это чертёжный кабинет, где настраивают всё для работы.

## Architecture Patterns

Паттерн 'Синглтон' используется в store для обеспечения единого источника данных, Паттерн 'Модульный компонент' реализован в каждом отдельном модуле (account, subscriptions и т.д.), Паттерн 'Фасад' применён в API-модуле для упрощения взаимодействия с внешними серверами

## Modules

- [api](modules/api.md) - Этот модуль отвечает за взаимодействие с внешними серверами через HTTP-запросы. Он обеспечивает авторизацию, управление данными компаний, получение информации о типах магазинов, обработку подписок, платежей и генерацию PDF-счетов. Модуль также содержит общие функции для безопасного взаимодействия с API, обработки ошибок и управления аутентификацией.

- [store](modules/store.md) - Этот модуль является центральным хранилищем состояния (Vuex) для Vue-приложения. Он объединяет различные модули (авторизация, подписки, аккаунт и т.д.), обеспечивая единый доступ к данным, логике изменений и отладке. Использует initialState для инициализации значений и предоставляет инструменты для управления состоянием в компонентах приложения.

- [company](modules/company.md) - Этот модуль отвечает за централизованное хранение и управление данными компании, лицензий и магазинов в приложении на Vue.js. Он использует Vuex для управления состоянием, взаимодействует с API для получения и обновления данных, а также предоставляет доступ к этим данным через getters. Модуль играет роль 'хранилища' для всех информации, связанной с компанией, и обеспечивает согласованность данных между различными частями приложения.

- [src](modules/src.md) - Этот модуль является основным файлом запуска Vue-приложения. Он инициализирует приложение, подключает необходимые библиотеки (например, axios для API-запросов, vue-axios для интеграции с HTTP-запросами), настраивает маршрутизацию, состояние (store) и стили. Также он обеспечивает совместимость с TypeScript через shims-vue.d.ts, что позволяет использовать Vue 3 с типами.

- [subscriptions](modules/subscriptions.md) - Этот модуль отвечает за управление подписками пользователей в приложении. Он хранит данные о тарифах, обрабатывает покупки, загружает информацию с сервера и предоставляет её другим частям приложения. Работает как центральный «хранилище» для всех данных, связанных с подписками, и обеспечивает их синхронизацию между интерфейсом и сервером.

- [auth](modules/auth.md) - Этот модуль отвечает за управление процессом авторизации пользователя в веб-приложении. Он хранит данные о состоянии аутентификации, взаимодействует с API для проверки учетных данных, и обеспечивает доступ к информации о пользователе. Работает на основе Vuex, используя действия (actions), мутации (mutations) и геттеры (getters) для управления состоянием. Подобен 'дверному замку' в здании: он проверяет, кто заходит, сохраняет информацию о посетителе и позволяет или запрещает доступ.

- [account](modules/account.md) - Этот модуль отвечает за централизованное управление данными аккаунта в приложении на Vue.js с использованием Vuex. Он загружает данные с сервера, хранит их в состоянии, позволяет обновлять информацию и предоставляет доступ к ней через геттеры. Аналогия: это как сейф в банке, где хранятся данные пользователя, и только определённые части системы могут к нему обращаться, чтобы получить или изменить информацию.

- [router](modules/router.md) - Этот модуль отвечает за управление навигацией в веб-приложении, защищая маршруты от неавторизованного доступа. Он использует vue-router для определения маршрутов и проверяет состояние пользователя через store, чтобы решать, куда перенаправлять. Также он использует разные layouts для разделения публичных и закрытых частей приложения, как будто это пропускной пункт в здании, где некоторые двери доступны только авторизованным людям.

- [utils](modules/utils.md) - Этот модуль содержит утилиты для работы с файлами, кодирования данных и генерации QR-кодов. Он помогает скачивать файлы из API, кодировать текст в специфические форматы и создавать временные задержки. Его используют, когда нужно обработать данные или подготовить их для передачи в другие системы.

- [wipon-pro-main](modules/wipon-pro-main.md) - Этот модуль содержит набор конфигурационных файлов, которые обеспечивают удобную и качественную разработку Vue-приложения с использованием TypeScript. Он отвечает за тестирование, линтинг, транспиляцию, управление переменными окружения и форматирование кода, создавая целостную среду разработки.

- [config](modules/config.md) - Этот модуль служит централизованным хранилищем настроек для приложения. Он управляет ценами, типами устройств и конфигурациями API, обеспечивая единый источник данных для других частей системы. Например, как центральный ящик в доме, где хранятся все ключи и настройки, доступные всем жителям.

## Data Flow

Данные приходят через API-модуль (как почта в дом), сохраняются в store (центральный сейф), затем используются модулями (как жители, берущие данные из сейфа). При необходимости данные обрабатываются в utils (мастерской), а затем возвращаются в интерфейс через компоненты.

## Key Components

- Store — центральное хранилище, как сейф, где хранятся все данные пользователя и компании
- API — почтовый ящик, обеспечивающий связь с внешними серверами
